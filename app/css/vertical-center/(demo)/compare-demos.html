方案 实现说明 优点 缺点 推荐指数
Flex 容器设置 display: flex，使用 justify-content 和 align-items: center • 代码简洁明了
• 容易理解和维护
• 完美居中
• 响应式自适应 		• IE11及以下支持有限 ⭐⭐⭐⭐⭐
行内块 使用 line-height和 vertical-align: middle 对齐 • 兼容性好
• 不需要固定宽高 		• 代码相对复杂
• 需要理解行内块对齐原理 		⭐⭐⭐
Table 使用 display: table 和 table-cell 布局 • 兼容性好
• 垂直居中稳定
• 不需要固定高度 		• 需要额外DOM结构
• 代码相对复杂 		⭐⭐⭐
Grid 使用 display: grid 和 place-items: center • 代码最简洁
• 功能强大
• 一行代码搞定 		• IE支持有限 ⭐⭐⭐⭐
Position + Transform 使用绝对定位和 transform 偏移 • 兼容性好
• 不需要额外标签 		• Transform可能影响子元素
• 需要理解transform原理 		⭐⭐⭐⭐
Margin Auto 使用绝对定位和 margin: auto • 兼容性好
• 概念简单 		• 需要明确的宽高
• 定位相对复杂 		⭐⭐⭐
伪元素 使用伪元素和 vertical-align: middle 对齐 • 兼容性好
• 不需要额外DOM结构
• 适合固定布局 		• 需要处理空白间隙
• 需要设置font-size: 0
• 页面缩放时需要注意间距 		⭐⭐⭐
最优选择（按推荐度排序）：
1. Flex 方案：现代网站的首选方案
2. Grid 方案：代码最简洁，但需要考虑兼容性
3. Position + Transform：兼容性好的通用方案
4. Table 方案：需要考虑 IE 时的备选方案
